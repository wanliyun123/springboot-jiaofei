

该系统目前是一个银行在线缴费系统的雏形，相关介绍如下：
    环境：mysql5.6 ；springboot2.4；session进行用户拦截；mybatis-generator生成sql语句
    数据库设计：目前设计的是三个系统，分别是useraccount，userpwd以及payrecord三张表，分别是
用户个人信息表：用于存储用户个人信息；
用户密码表：用于存储用户id和密码（注意这里的id是和useraccount中的userid相关联的外键）；
payrecord表：用于存储当用户提交缴费信息后，存储该缴费信息的一张表。

	1.主页：template/login页面。在浏览器界面输入localhost：8080进入主页，登陆用户名和密码
	2.list页面：template/list。输入正确的用户id和密码后，进入个人信息页面，相关的个人信息从
	   useraccount表中拿出来个人信息以及账户余额，水电话费余额展示在个人信息页面
	3.水电费充值：template/addmoney。点击充值按钮，来到提交页面，涉及到充值金额，充值类型选择
	4.处理充值请求：用户提交充值请求，先判断余额是否足够充值，不足充值，返回list页面，并显示
	   余额不足信息。余额足够根据充值类型充值到指定的水电费上，并将余额更新到list页面。
	5.历史账单查询： 点击历史账单，发送历史账单请求，同时带上用户id，mybatis通过where userid = userid语句
	  得到所有该用户下的账单，返回到历史账单界面。
	6.目前该系统的功能暂时只是这些，一个完整的系统应该添加一些安全验证，比如ip验证，对于后台管理页面，只允许
	  服务器内网ip进行访问，当外网想要登陆admin账号时，拦截该ip所发出的请求。
	7.用户登陆时可以获取对方的IP地址，记录下常用ip地址，如果有一场ip登陆时，进行验证码登陆。
	8.当用户水电费某一项欠费时，对其发送短信息进行提醒。
	9.用户每失信逾期一次，信用等级下降一级，多次逾期或长期不缴纳将拒绝使用其中的优惠功能，知道缴费成功保持到
	  信用等级上升。同时用户长期缴纳费用，可以提升信用等级，根据其消费水平进行相应的优惠推送。
